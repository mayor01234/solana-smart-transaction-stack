# Evidence

Generated by `npm run challenge:first-place` against **live** infrastructure (`ALLOW_DRY_RUN=false`).
Judges can cross-reference the slot numbers and signatures here on a Solana explorer.

| File | What it is |
|---|---|
| `lifecycle-log.json` | Full array of bundle lifecycle records (primary submission artifact). |
| `lifecycle-log.jsonl` | Same records, one JSON object per line (stream-friendly). |
| `lifecycle-log.md` | Human-readable table of records. |
| `run-summary.json` | Aggregate counts, unique tip values, and p50/p90 latency deltas. |
| `scorecard.json` | First-place scoring-gate result. |
| `verification-report.md` | Pass/fail of the evidence against the quality policy. |
| `lifecycle-log.example.json` | Illustrative example only — **not** valid submission evidence. |

Each record contains: bundle id + signatures, submitted/processed/confirmed/finalized slots and
timestamps, latency deltas, the dynamic tip and tip account, the AI decision trace (action + tip +
reasoning), any failure classification, the commitment source per stage, and explorer links.

> Only evidence generated with live infrastructure is valid. Dry-run logs are for local development
> and must never be submitted.
